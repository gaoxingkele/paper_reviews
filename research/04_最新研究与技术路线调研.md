## 0. 起点：OpenAIReview / OpenReviewer 是什么

本次调研的触发对象其实是**两个同名易混的项目**，需先区分清楚：

### 0.1 OpenAIReview（ChicagoHAI）— 一个"流水线 + 多模型"的开源审稿工具
- **是什么**：芝加哥大学 Chicago Human+AI Lab (ChicagoHAI) 出品的开源自动审稿系统，目标是"用一杯咖啡的成本拿到高质量审稿意见"，并提供 Web 界面与命令行（`pip install openaireview`），同时打包成 **Claude Code 技能**（有 Claude Code 即可免费用）。
- **系统设计**：同时调用 **Claude 与 Gemini 多模型**对论文做评审；支持 **PDF 与 TeX** 双格式输入；采用"running summary（边读边摘要）+ 逐段（passage-by-passage）分析"的流水线；自带 **Refine benchmark** 把 AI 审稿与人类标注对比。
- **出处**：项目主页 https://openaireview.org/ （原 https://openaireview.github.io/ 已 301 重定向）；GitHub https://github.com/ChicagoHAI/OpenAIReview ；立场博客《AI-assisted Reviewing is Necessary and Should be Open》 https://openaireview.org/blog.html

### 0.2 OpenReviewer（maxidl）— 一个"专用微调小模型"
- **是什么**：把"审稿能力"蒸馏进一个 8B 专用模型 **Llama-OpenReviewer-8B**（基于 Llama-3.1-8B-Instruct 全参微调，128k 上下文）。
- **数据与训练**：ICLR/NeurIPS 2022–2025 的约 **79,000 条专家审稿 / 36,000 篇论文**；用 **Marker** 把 PDF 转 markdown（含公式表格），按置信度与长度过滤；3 epoch，64×A100 训 34 小时。
- **关键技巧（对标定很重要）**：通用 LLM 给分系统性偏高（GPT-4o 平均 6.9–8.1 分），而 OpenReviewer 平均 **5.4 分恰好对齐人类分布**；Exact-Match 命中率 **55.5% vs GPT-4o 23.8%**，平均绝对误差 **0.96 vs 2.34**。按会议模板（ICLR 默认）输出结构化审稿（summary/strengths/weaknesses/score）。
- **出处**：《OpenReviewer: A Specialized Large Language Model for Generating Critical Scientific Paper Reviews》arXiv:2412.11948 ；GitHub https://github.com/maxidl/openreviewer ；HuggingFace Space demo。

> **启示**：这两条路线（"多模型流水线" vs "专用微调小模型"）正是当前两大技术分叉。OpenAIReview 在第三方基准里是最强配置之一（见 §3）。

## 1. 多智能体审稿框架（角色扮演 / 辩论 / 讨论）

### 1.1 AgentReview — 首个 LLM-Agent 同行评审"全流程仿真"
- **设计**：三类角色 Agent——**Reviewer / Author / Area Chair (AC)**，跑 **五阶段流水线**：①独立评审 → ②作者 rebuttal → ③Reviewer-AC 讨论并改分 → ④AC 写 meta-review → ⑤最终决策。可给 Reviewer 注入特质（负责/不负责、善意/恶意、知识水平），给 AC 注入决策风格（专断/从众/包容）。
- **核心价值**：不是为了"代替审稿"，而是**解耦并量化偏见**：37.1% 决策差异源于 reviewer 偏见；rebuttal 后社会影响使评分标准差降 27.2%（从众效应）；暴露作者身份会翻转 27.7% 的决策。
- **出处**：《AgentReview: Exploring Academic Peer Review with LLM Agent》(EMNLP 2024 Oral) arXiv:2406.12708 ；GitHub https://github.com/Ahren09/AgentReview

### 1.2 ReViewGraph — "审稿人-作者辩论" + 异构图推理
- **设计**：三阶段（3 个 reviewer agent 初评 → author agent 逐点 rebuttal → reviewer 重评），用 Qwen2.5-VL-72B 做多模态（文本+图表）。**亮点是把辩论过程建成异构图**：节点=论文/评价维度/审稿意见/作者意见；边=审稿人之间关系（同意/反对/互补/递进/独立）与审稿人-作者交互（接受/拒绝/澄清/妥协/扩展/中立），再用 **Heterogeneous Graph Transformer (HGT)** 做关系推理。
- **为什么强**：能识别"措辞礼貌但整体偏负面"的隐性共识、避免被孤立的少数派批评带偏；**无需微调**即在 ICLR 2023/2024/2025 达 70.3%/66.1%/71.7% 准确率，比次优基线（CycleReviewer-70B）平均相对提升 15.7%。
- **出处**：《Automatic Paper Reviewing with Heterogeneous Graph Reasoning over LLM-Simulated Reviewer-Author Debates》arXiv:2511.08317 ；GitHub https://github.com/relic-yuexi/ReViewGraph

### 1.3 MARG — 多 Agent 内部讨论 + 长文分片
- **设计**：多个 LLM 实例分工讨论：**leader（协调）+ worker（分配到论文不同段落，从而突破上下文长度限制读完全文）+ expert（针对 experiments/clarity/impact 等子任务专精）**。
- **效果**：把 GPT-4 的"泛泛评论"比例从 60% 降到 29%，每篇产出 3.7 条有效意见（2.2 倍提升）。
- **出处**：《MARG: Multi-Agent Review Generation for Scientific Papers》arXiv:2401.04259 (D'Arcy et al., AllenAI) ；GitHub https://github.com/allenai/marg-reviewer

### 1.4 MAMORX — 多智能体 + 多模态 + 外部知识
- **设计**：专门 agent 分别评估 novelty、figure 批评、clarity，融合**图表/表格/引用等多模态输入**并接入外部知识源。
- **效果**：对人类审稿的估计胜率约 93%，优于次优多 agent 基线。
- **出处**：《MAMORX: Multi-agent Multi-Modal Scientific Review Generation with External Knowledge》(NeurIPS 2024) OpenReview id=frvkE8rCfX ；GitHub https://github.com/sciosci/mamorx-review-system ；demo https://rev0.ai

### 1.5 ScholarPeer / Stanford Agentic Reviewer — 上下文感知多 Agent（已产品化）
- **设计**：Summarizer + Review-Generator 等多 agent；强调**上下文感知**——多层 web 检索查询（benchmarks/baselines、同问题论文、相关技术论文，经 Tavily API），相关工作排序后下载 arXiv PDF→markdown→LLM 详摘，再按七维（originality / importance / claim support / experimental soundness / clarity / community value / contextualization）打分，用线性回归映射到总分做**校准**。
- **可靠性数据**：与人类 Spearman 相关 **0.42（人-人为 0.41）**；接收预测 AUC 0.75（人类 0.84）；分数分布与人类对齐良好。
- **出处**：《ScholarPeer: A Context-Aware Multi-Agent Framework for Automated Peer Review》arXiv:2601.22638 ；产品页/技术总览 https://paperreview.ai/tech-overview

## 2. 顶会/大型会议在用或试点的 AI 辅助审稿

### 2.1 AAAI-26 AI Review Pilot — 迄今最大规模实测（22,977 篇）
- **做法**：**每篇主赛道论文都附 1 条明确标注的 AI 审稿**，与 ≥2 条人类审稿并列（Phase 1）。流水线为 **五个专门阶段**：story（问题）、presentation（清晰度）、evaluations（方法）、correctness（公式/算法，**调用代码解释器验证**）、significance（新颖性/定位，**调用 web 搜索**）；外加 **self-critique 自我批评 + 人工抽检**（防引用幻觉、防泄露作者身份、查伦理）。
- **成本/规模**：GPT-5，24 小时内跑完全部论文，约 **$1/篇**。引用核查：100 篇样本 1,346 条引用，仅 2 条存疑。
- **结果**：5,834 份问卷显示 AI 审稿在 9 项质量标准中的 6 项优于人类；技术准确性 +0.67；53.9% 认为有用；作者最看重其"可操作的修改建议"与"广度与全面性"。**短板**：新颖性/科学重要性判断弱、啰嗦/纠缠小问题、偶尔读错数学记号。
- **出处**：《AI-Assisted Peer Review at Scale: The AAAI-26 AI Review Pilot》arXiv:2604.13940

### 2.2 ICLR 2025 — "审稿反馈 Agent"（辅助审稿人而非代替）
- **做法**：用多 LLM 给**审稿人**的审稿稿提供可选反馈，使其更 informative/clear/actionable；**明确不替代人类、不自动写/改审稿**。动机是投稿量暴涨（ICLR 2024/2025 同比 +47%/+61%）。
- **结果**：观察到审稿在 specificity、engagement、actionability 上提升；会后向作者/审稿人/AC 发匿名问卷评估。
- **出处**：ICLR 官方博客《Leveraging LLM feedback to enhance review quality》 https://blog.iclr.cc/2025/04/15/leveraging-llm-feedback-to-enhance-review-quality/ ；《Assisting ICLR 2025 reviewers with feedback》 https://blog.iclr.cc/2024/10/09/iclr2025-assisting-reviewers/

### 2.3 出版社政策（Springer Nature / Elsevier）— "可辅助、需问责、难强制"
- **要点**：LLM 不满足作者署名标准；审稿人对其报告的准确性与观点**负全责**，同行评审建立在互信之上；生成式 AI 有知识过时、可能产出错误/偏见信息的局限；政策持续监测更新。
- **政策可执行性质疑**：《Policies Permitting LLM Use for Polishing Peer Reviews Are Currently Not Enforceable》arXiv:2603.20450 —— 测了 5 个 SOTA 检测器，均把相当比例"LLM 润色过的人类审稿"误判为 AI 生成（高假阳），即便加入稿件访问与领域知识信号也达不到判定标准；结论是**靠检测来执行"仅允许润色"的政策当前不可行**，公开的"AI 使用率"估计可能被高估。
- **出处**：Springer 政策 https://www.springer.com/gp/editorial-policies/artificial-intelligence--ai-/25428500 ；arXiv:2603.20450

## 3. 第三方基准：哪些路线"真的可靠"

### 3.1 Benchmarking Agentic Review Systems — 系统级横评
- **对比对象**：zero-shot 单提示 / **OpenAIReview**（running summary + 逐段）/ 某 coarse 多 agent（section 级）/ Reviewer3（商业闭源）；跑在 GPT-5.5、Claude Opus 4.7、Grok-4.1-Fast 等 6 个后端。
- **两套评测法**（值得借鉴）：①**质量信号相关性**——评论量是否随论文质量变化（用引用/接收/评分等外部信号），指标 pairwise accuracy；②**扰动基准**——往论文注入 4 类错误（数学、虚假声明、错误推理、实验缺陷），测 recall。
- **关键结论**：**OpenAIReview + GPT-5.5 最强**，质量区分 pairwise 准确率 83.0%；最佳系统抓到 71.6% 注入错误，**多模型并集达 83.3%（不同模型互补）**；用户点赞:点踩 = 1.44:1，被踩多为"无用/小题大做/假阳"；人类强在新颖性、AI 强在技术细节，**二者互补**。
- **出处**：《Benchmarking Agentic Review Systems》arXiv:2606.19749

## 4. 提升可靠性的技术（对抗核验 / 投票 / RAG 引用核查 / rubric / 与人对齐）

### 4.1 DeepReview — 把"人类深度思考"显式拆成多阶段
- **三阶段推理**：①**Novelty Verification**（语义检索约 60 篇→排前 10 篇判原创性）→ ②**Multi-Dimension Evaluation**（从作者 rebuttal 反推批评、把批评转成具体技术建议）→ ③**Reliability Verification**（系统性证据分析，给每条意见赋置信度）。
- **模型/数据**：DeepReviewer-14B（基于 Phi-4），三档推理模式（Fast/Standard/Best）支持**测试时算力伸缩**；数据集 DeepReview-13K（13,378 篇，带结构化推理链）。
- **效果**：相对 CycleReviewer-70B 把评分 MSE 降 44.8%；对 GPT-o1 胜率 88.21%；**未做对抗训练却对对抗攻击更鲁棒**。
- **出处**：《DeepReview: Improving LLM-based Paper Review with Human-like Deep Thinking Process》arXiv:2503.08569 ；GitHub https://github.com/zhu-minjun/Researcher

### 4.2 FactReview — 证据接地 + 文献定位 + **执行式声明核验**
- **设计**：抽取审稿相关 claims → 用相关工作 ground（文献定位）→ **有代码就跑 released artifacts 验证经验性声明**。这是把"RAG 引用核查"升级到"真正执行复现"。
- **效果**：35 篇 ML 论文覆盖 84% claims，证据感知评分 4.86/5；**执行证据改变 17% 的 claim 判定（单一最强证据源）**；辅助人类审稿时完成时间降 58%，基准声明覆盖从 87%→99%。
- **核心立场**：**LLM 审稿应"审计经验性声明"，而非做接收/拒绝决策**。
- **出处**：《FactReview: Evidence-Grounded Reviews with Literature Positioning and Execution-Based Claim Verification》arXiv:2604.04074（代码已开源）

### 4.3 Reasoning LLM 做"关键问题检查器" + LLM-as-Judge 自动评测
- **思路**：定位"会推翻论文结论的关键错误/不严谨"，而非生成完整审稿；输入对比 PDF/OCR/LaTeX 三种（**LaTeX 保留数学记号最稳**）。
- **自动评测**：用多个 reasoning 模型做 LLM-as-judge，对照作者提供的 gold 错误描述，用 Hit Rate@k / AP@k 衡量。**o3 性价比最高**；o 系列在 PDF↔LaTeX 间稳定，Gemini 用 LaTeX 掉点，Claude 3.7 Sonnet 吃 PDF 较弱（仅 35% 找到问题）。
- **出处**：《Reviewing Scientific Papers for Critical Problems With Reasoning LLMs: Baseline Approaches and Automatic Evaluation》arXiv:2505.23824

### 4.4 CycleResearcher / CycleReviewer — 把审稿当"奖励模型"做研究-评审-精炼闭环
- **设计**：CycleResearcher（策略模型写论文）+ CycleReviewer（奖励模型模拟多审稿人打分）做 **Research-Review-Refinement** 迭代。
- **效果**：CycleReviewer 预测论文分数的 MAE 比单个人类审稿低 26.89%。**伦理护栏**：>95% 准确的 AI 生成检测器 + 输出水印 + 披露声明；数据集 Review-5k / Research-14k 开源。
- **出处**：《CycleResearcher: Improving Automated Research via Automated Review》(ICLR 2025) arXiv:2411.00816

### 4.5 ReviewRL — 用强化学习训练审稿模型
- **思路**：把"自动科学审稿"建成 RL 问题（结合检索增强上下文与可验证奖励），提升评分准确性与意见质量。
- **出处**：《ReviewRL: Towards Automated Scientific Review with Reinforcement Learning》arXiv:2508.10308

### 4.6 立场/伦理：把 AI 定位为"导师/反馈"以补审稿人缺口
- 《Position on LLM-Assisted Peer Review: Addressing Reviewer Gap through Mentoring and Feedback》arXiv:2601.09182 —— 主张用 AI 给（尤其新手）审稿人做 mentoring/feedback，缓解审稿人荒，而非取代。
- 现实警示：ICLR 2026 被报有约 **21% 审稿疑似 AI 生成**，引发诚信争议 —— 政策与检测的张力（呼应 §2.3 不可执行性）。出处：WebProNews 报道 https://www.webpronews.com/iclr-2026-scandal-21-of-peer-reviews-ai-generated-raising-integrity-issues/

## 6. 针对"特定期刊/学科定制化审稿"的可行做法（综合各源归纳）
1. **会议/期刊模板驱动**：OpenReviewer、Stanford Agentic Reviewer 都按目标会议的 review template/七维 rubric 输出 —— 换期刊即换 rubric/checklist 模板（用户可自定义模板）。
2. **rubric→分数的校准回归**：Stanford Agentic Reviewer 用线性回归把维度分映射到该期刊的总分尺度，使分布对齐目标刊物。
3. **学科水平匹配靠检索**：用 web/arXiv 检索同问题/同方法论文做"文献定位"，让新颖性判断贴合该领域现状（DeepReview、ScholarPeer、FactReview）。
4. **分数去偏校准**：针对通用 LLM 给分偏高，用专用数据微调（OpenReviewer 对齐人类 5.4 均分）或后处理校准。

## 7. 总结：当前主流技术路线谱系
- **路线 A：多模型流水线 + 逐段分析**（OpenAIReview）—— 工程简单、可叠多后端投票，第三方横评最强配置之一。
- **路线 B：专用微调小模型**（OpenReviewer-8B / DeepReviewer-14B / CycleReviewer）—— 低成本、分数对齐人类、可控。
- **路线 C：角色扮演多 Agent + 辩论/讨论**（AgentReview / ReViewGraph / MARG / MAMORX）—— 接近真实评审动力学，ReViewGraph 进一步用异构图把辩论结构化。
- **路线 D：证据接地与执行核验**（FactReview 执行代码、DeepReview 检索核验、reasoning-LLM 查关键错）—— 把"可靠性"做实，定位为"声明审计员"而非"判决者"。
- **路线 E：会议级真实部署**（AAAI-26 全量、ICLR 反馈 Agent）—— 共识是 **AI 辅助、人类问责、互补而非替代**。
