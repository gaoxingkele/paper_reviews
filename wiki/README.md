# 思维演化 Wiki — paper_reviews

记录本项目"方法/思维的进化"：为什么引入某个思想、它解决什么根本问题、我们怎么改造它、
被谁取代/取代了谁、怎么移植到别处。区别于 `research/`（来源事实分析）与 commit（操作记录）。

每篇必须标注引用出处；用 `[[条目名]]` 互链思想谱系；结尾给"移植提示"。

## 方法论说明
- 引入外部思想并据此改了架构 → 新增/更新一篇
- 一次有意义的方法论演化（落地或被否但有元教训）→ 记一篇
- 整体策略框架迁移 → 记一篇
- 纯操作改动（bugfix/重命名）→ 不写 wiki，只走 commit
